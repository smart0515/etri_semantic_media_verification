# 검증 절차서

## 1. 사전 준비

1. 검증 대상 버전과 범위를 동결한다.
2. 데이터셋 사용 승인·라이선스·미디어 권리를 확인한다.
3. annotation·영상·설정·모델 파일의 SHA-256을 기록한다.
4. 시험 ID, 실행 ID, 담당자, 시간, 환경 ID를 발급한다.
5. 성공·실패·제외 판정 기준을 ETRI와 합의한다.

## 2. 실행

1. 입력 adapter로 annotation을 공통 모델로 변환한다.
2. 검증시스템과 ETRI 대상 시스템에 동일 비디오·query를 입력한다.
3. 분석, 전송, 재조립, 재생 단계별 시작·종료 시각과 출력을 수집한다.
4. 시스템 출력을 `prediction.schema.json`으로 정규화한다.
5. 정답과 예측을 qid·vid로 매칭한다.
6. 구간, 하이라이트, element, relation, 재조립, 성능 지표를 산출한다.
7. 원시 출력, 정규화 출력, 지표 결과, 로그의 hash를 저장한다.

## 3. 판정

- `PASS`: 합의된 기준을 충족하고 중대 오류가 없음
- `FAIL`: 기준 미달 또는 필수 단계 실패
- `CONDITIONAL_PASS`: 핵심 기능은 충족하나 후속 조치가 필요
- `NOT_TESTED`: 환경·데이터·연동 미준비로 미수행
- `EXCLUDED`: 승인된 범위 제외

현 보고 단계는 `PRELIMINARY_MOCK` 상태이며 실제 ETRI 판정을 생성하지 않는다.

## 4. 이상 처리

1. 입력 결함, 시스템 오류, 평가기 오류, 환경 오류로 분류한다.
2. 재현 절차와 최소 재현 입력을 남긴다.
3. 수정 전·후 버전과 결과를 연결한다.
4. 사전 합의 없이 기존 결과를 덮어쓰지 않는다.

## 5. 종료 및 증빙

- 시험결과서
- 시나리오별 원시 로그
- 평가 결과 JSON/CSV
- 화면 증빙 또는 재생 확인 기록
- 환경 명세·파일 hash·버전 목록
- 이슈·조치 대장
- 상호 입회 회의록·확인서
