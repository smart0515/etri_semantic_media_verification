# 검증 절차서

## 1. 사전 준비

1. 대상 시스템, 모델, 데이터셋, 설정 버전을 식별한다.
2. 데이터와 소프트웨어의 이용·배포 조건을 확인한다.
3. 입력 형식, 시간 단위, clip 간격과 오류 계약을 고정한다.
4. 시험 시나리오, 표본, 지표, 기준값과 제외 항목을 승인한다.
5. 실행 ID와 증빙 저장 경로를 생성한다.

## 2. 실행 절차

1. 원시 입력과 입력 manifest의 hash를 기록한다.
2. 동일 입력을 참조 시스템과 대상 시스템에 제공한다.
3. 원시 출력을 변형 없이 보존한다.
4. adapter를 사용해 공통 Prediction 계약으로 정규화한다.
5. Moment, Highlight, Element, Relation, Reassembly 지표를 계산한다.
6. 스키마, relation 참조와 결과 파일 간 qid 일치 여부를 검사한다.
7. 환경정보, 로그, 지표, 이슈와 실행 시간을 저장한다.
8. 기준값에 따라 PASS, CONDITIONAL_PASS, FAIL 또는 EXCLUDED를 부여한다.
9. 조치가 필요한 항목은 원인, 담당, 기한과 재시험 결과를 연결한다.
10. 검증 참여자가 결과표와 증빙 목록을 확인한다.

## 3. 증빙 항목

- 소스 commit 및 configuration hash
- dataset 명칭·버전·입력 hash
- 대상 시스템·모델·adapter 버전
- 원시 출력과 정규화 prediction
- 지표 JSON과 재조립 계획
- 실행 환경, 시작·종료 시각 및 로그
- 이슈 목록, 조치 및 재시험 결과

## 4. 품질 관리

- 필수 필드와 자료형은 JSON Schema로 검사한다.
- 대표 계산 예제를 단위시험으로 고정해 산식 변경을 감지한다.
- 결과 JSON과 보고서 표를 동일 실행 결과에서 생성한다.
- 비밀정보와 원본 미디어 경로는 결과 패키지에 직접 기록하지 않는다.
